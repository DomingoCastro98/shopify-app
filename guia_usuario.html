Shopify Utilidades App · Guía de usuario

Un panel local para Shopify, Docker y temas, sin fricción innecesaria.

Esta app centraliza la gestión de contenedores, el flujo de importación/exportación de themes, los perfiles de trabajo, los logs y la documentación operativa. La intención de esta guía es que puedas entender el flujo completo en pocos minutos y volver a él cuando algo falle.

Qué puedes hacer aquí

  • Crear, recrear, detener y administrar contenedores Shopify desde una sola interfaz.
  • Importar themes desde .zip o carpeta, con validación previa y backup automático.
  • Exportar el trabajo local o sincronizar múltiples contenidos remotos con un flujo guiado y multiselección.
  • Listar y descargar temas remotos desde Shopify, con opción de marcarlos como activos directamente.
  • Gestionar volumes y networks en Docker sin fricciones, con protección automática de datos críticos.
  • Acceder al contenedor por Remote-SSH con un modal de progreso y detección de configuración existente.
  • Usar F1 para abrir esta guía desde la app.

1Qué hace esta aplicación

Shopify Utilidades App es una interfaz de escritorio en Python para trabajar con temas Shopify dentro de contenedores Docker. La app evita instalar de forma global Node.js, Ruby o Shopify CLI en el sistema, y agrupa el flujo de trabajo en una sola ventana.

Contenedores

Ver estado, arrancar, parar, borrar, renombrar y abrir acceso remoto por SSH con flujo guiado.

Theme workflow

Importar, validar, auto-corregir settings, crear backup previo y opcionalmente ejecutar push o pull remoto.

Visibilidad

Panel de estado, historial de acciones, logs y documentación accesible desde la barra lateral y F1.

2Inicio rápido

1

Comprueba Docker

La app necesita Docker Desktop o un daemon Docker accesible. El panel lateral te mostrará si Docker está disponible y qué modo está activo.

2

Elige el modo correcto

En local, la app trabaja contra Docker en tu máquina. En remoto, usa un host Docker TCP o SSH configurado por la propia aplicación.

3

Abre el flujo que necesitas

Usa la barra lateral para crear/recrear entorno, importar, exportar o administrar contenedores. Cada flujo abre un asistente guiado.

4

Valida antes de ejecutar

Los asistentes de importación y exportación incluyen resumen de configuración y validación previa. En importación puedes activar backup automático antes de modificar nada.

Consejo: si no recuerdas qué hace una sección, pulsa F1 para volver a esta guía.

3Funciones principales

Crear / Recrear entorno

Inicia un contenedor limpio con dependencias de trabajo y, si hace falta, vuelve a construir el entorno desde cero. Es la opción adecuada cuando quieres partir de una base nueva o cuando el contenedor actual ya no es fiable.

Gestionar contenedores

Incluye listado, arranque, parada, eliminación, renombrado, inspección rápida y acceso remoto SSH para trabajar dentro del contenedor.

Gestionar volumes y networks

Crea, inspecciona, clona y limpia volumes. Conecta y desconecta contenedores de networks con protección automática de volumes críticos (incluyendo *_history_remote y *_profiles_remote) en operaciones prune.

Acceso remoto SSH

Al pulsar Acceso Remoto (SSH), la app muestra un modal con progreso en vivo. Si ya existe configuración en ~/.ssh/config, permite editar HostName y Port; si no cambias nada, reutiliza la configuración y abre VS Code directamente.

Temas remotos

Lista los temas disponibles en tu tienda Shopify. Puedes seleccionar múltiples temas y descargarlos de una vez. Al importar, tienes la opción de marcar un tema como activo para cambiar el tema actual.

Perfiles

La app guarda conjuntos de contenedores para agrupar entornos de trabajo. Puedes mantener perfiles privados o remotos según el modo de conexión.

Historial y logs

Toda acción relevante queda registrada. El historial se puede filtrar por nivel y búsqueda textual, y los logs se exportan o copian desde la UI.

4Importación y exportación

El flujo de importación y exportación ahora soporta multiselección remota y mayor control visual. Todas las operaciones largas usan un modal unificado de "Procesando..." con spinner animado y estado final.

Importar theme / datos

  • Selecciona una carpeta o un .zip.
  • Pulsa Validar para ver el resumen de archivos y la preparación del flujo.
  • Activa el backup automático si quieres una copia previa del contenedor.
  • La app ejecuta npm install y, si procede, el auto-fix de settings.
  • Opcionalmente marca como activo para cambiar el tema actual después de la importación.
  • Si eliges push, se abre el proceso guiado para publicar en Shopify.

Exportar theme / datos

  • El modo local empaqueta el contenido del contenedor y lo extrae en una carpeta de destino.
  • El modo remoto te permite listar y seleccionar múltiples temas desde Shopify.
  • Los temas remotos se descargan en paralelo con un modal de progreso.
  • También hay validación previa para no lanzar una exportación mal configurada.
  • Los resultados se guardan en una carpeta con marca temporal para evitar sobrescrituras accidentales.
A

Flujo local vs remoto

Cuando abres el exportador, puedes elegir entre Local (contenedor en tu máquina) o Remoto (Shopify CLI). En remoto, la app lista automáticamente los temas disponibles con multiselección.

B

Modal de progreso

Durante cualquier operación larga (descargas, importaciones, exportaciones), verás un modal con spinner animado. El estado final muestra completado (verde) o error (rojo) con opción de cerrar manualmente.

C

Después de importar/exportar

Si es una importación y marcaste "activo", el tema se establece como actual sin pasos adicionales. Los logs muestran el resumen de operación.

Importante: si activas backup automático, la app crea una copia del contenido del contenedor antes de tocar el theme. Es la opción recomendada cuando vas a hacer cambios grandes.
Protección de datos: en las operaciones de limpieza (prune) de volumes, la app detecta automáticamente y protege todos los volumes que contengan _history_remote o _profiles_remote en el nombre, independientemente de la tecnología. Nunca se borrarán accidentalmente.

6Solución de problemas

F1 no abre la guía

La app busca guia_usuario.html en la carpeta raíz del proyecto y, si falla, intenta abrir README.md como fallback.

Docker no aparece disponible

Verifica que Docker Desktop esté iniciado y que el modo local o remoto coincida con tu configuración. El panel lateral te mostrará el estado detectado.

Importación rechazada por validación

Comprueba la ruta del theme, si el ZIP está intacto y si el contenedor seleccionado es el correcto. El resumen de validación te dirá qué falta.

Exportación remota no avanza

Normalmente el problema está en la sesión de Shopify CLI, en la URL de tienda o en credenciales no válidas. Revisa la terminal que se abre para el login.

Modal "Procesando..." se queda colgado

Las operaciones largas se ejecutan en un hilo secundario. Si se cuelga, revisa los logs y considera reiniciar la app. El modal siempre te permite cerrar manualmente.

Acceso remoto SSH parece detenido

El flujo actual muestra un modal con pasos en vivo (sshd, clave pública, puerto y ~/.ssh/config). Si tarda, espera a que termine la instalación inicial de openssh dentro del contenedor.

Prune de volumes rechazado

El sistema detecta automáticamente y protege volumes con _history_remote y _profiles_remote en el nombre. También se protegen todos los volumes en uso. Solo se eliminan los realmente sin referencia.

7Buenas prácticas

  • Antes de importar, valida siempre el paquete y activa backup automático cuando el cambio sea importante.
  • En exportación remota, usa multiselección para descargar varios temas de una sola vez. El modal de progreso te indicará el estado.
  • Marca temas como "activos" al importar remotamente solo si tienes certeza de que quieres cambiar el tema actual de la tienda.
  • En gestión de volumes, el sistema protege automáticamente *_history_remote y *_profiles_remote en prune. No necesitas preocuparte.
  • Usa perfiles para agrupar contenedores por proyecto o por entorno.
  • Mantén el historial limpio y usa filtros para encontrar rápidamente una acción concreta.
  • Usa Acceso Remoto (SSH) como vía principal de edición remota; ya no depende de editor web en el contenedor.
  • Cuando trabajes con varios monitores o pantallas pequeñas, activa la vista compacta con Ctrl + B.
  • Usa F1 como acceso rápido a esta guía cuando no recuerdes el flujo correcto.
Idea práctica: si algo ya funcionaba y dejó de funcionar, primero revisa el modo Docker y el panel de estado. Las operaciones largas ahora tienen modal visual, así que no dudes en esperar si el spinner sigue animado.
